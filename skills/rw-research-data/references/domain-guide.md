# RW Research Data 选择表

| 情形 | 执行动作 | 不要做 |
|---|---|---|
| 材料可以公开 | 检查持久标识符、版本、许可证、元数据和实际下载路径 | 只贴项目主页 |
| 材料需要受控访问 | 记录申请入口、条件、审批方和预计处理方式 | 写 available on request 但没有流程 |
| 材料不能提供 | 记录限制依据、可公开元数据和可替代材料 | 编造公开链接 |
| 代码依赖特定环境 | 记录版本、依赖和运行入口 | 只上传单个脚本 |
| 标识符尚未生效 | 标记待确认并安排提交前复核 | 写成已发布 |

## 核心规则

- 公开、可取得、可互操作和可复用是不同状态，分别核验。
- 受控访问材料可以满足可取得性，但要有可执行的申请入口和审批规则。
- 只有仓库首页、项目主页或个人网盘链接，不能证明具体版本可取得。
- 持久标识符要指向明确对象和版本，不能用论文 DOI 代替数据或代码标识符。
- 数据不能公开时仍要保存可公开元数据、限制理由和联系路径，除非这也违反规则。
- 许可证、伦理同意和数据使用协议分别限制复用，不能互相替代。
- 分析代码、运行环境和依赖版本属于复现路径，不能只列原始数据。
- 原始数据、处理后数据、派生变量、合成数据和示例数据要分开标记。
- 尚未签发的 DOI、accession 或禁运结束日期标记为待确认，不写成已生效。
- 访问测试只证明测试时点的路径状态，不证明材料质量或长期可用。
- 数据声明中的每个对象要能回到对象台账，不能用一句通用声明覆盖不同限制。
- 期刊、资助方和机构要求会变化，提交前核验当前官方页面。
- 敏感数据不上传到未经批准的服务，也不为测试绕过访问控制。
- 找不到材料时保留缺失状态和责任人，不生成文件、链接、许可证或批准记录。
