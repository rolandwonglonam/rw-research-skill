# 公理

## AXIOM-01：先固定基础版本

没有基础文稿 hash 时，不开始局部修改。

## AXIOM-02：未涉及内容不重写

Patch 只替换批准块，其他块正文保持原样。

## AXIOM-03：全部验证后再应用

任意 precondition 失败时整批停止，不生成部分修改稿。

## AXIOM-04：结构修改单独确认

新增、删除和重排章节需要用户明确确认，不能隐藏在段落替换中。

## AXIOM-05：范围控制不等于内容正确

Patch 证明修改边界，不证明新内容、数字或引用正确。

## AXIOM-06：批注位置不等于论证边界

先确定理解和修复问题所需的完整论证单位，再在用户批准的块内生成 Patch。需要跨块时先请求扩大范围。
