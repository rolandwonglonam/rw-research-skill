# 验收

- `SKILL.md` 前置字段只有 `name` 和 `description`。
- 至少 10 条知识原子、5 条公理和 3 个行为测试。
- `anchor` 不修改原始文件。
- manifest 包含文稿 hash、块 ID 和块 hash。
- `check` 能发现基础 hash、块 hash和重复块错误。
- `apply` 先验证全部操作，再生成新文件。
- report 包含修改块、保留块、保留比例和新文稿 hash。
- 修改比例超过 60% 时默认停止。
- 批注框选范围不被当作论证边界；报告说明实际修改单位及其理由。
- Explain、Expand、Tell me more 和 I don't understand 类批注不能只以增加字数、例子或同义词关闭。
- 对解释类批注，读者能从修改块识别当前功能所需的主体、对象、行动、发生环节、作用关系、适用边界和意义。
- 论证单位超出已批准块时停止应用并请求扩大范围，不静默越界。
