# RW PhD Write 选择表

| 情形 | 执行动作 | 不要做 |
|---|---|---|
| 只有主题没有来源 | 先建立论证和检索缺口 | 生成带引用正文 |
| 已有来源卡片 | 先做 function-to-claim-to-source 对齐 | 按文献逐篇总结 |
| 不知道某章承担什么功能 | 根据标题、前后文、研究问题和段落内容判断 | 根据 Chapter 编号套模板 |
| 连续引用研究但没有 takeaway | 提取共同点、冲突和边界，判断是否需要解释其意义和研究问题连接 | 只改语气或增加过渡词 |
| 导师写 Tell me more 或 Expand | 区分缺少研究细节、作者解释、逻辑连接还是研究问题关系 | 统一扩写整段 |
| 导师写 I don't understand | 检查指代、概念、前提和读者被要求自行补出的推理 | 只替换同义词 |
| 导师标出正向样本 | 提取功能、证据密度、解释深度和句间连接 | 只提取词汇和句长 |
| 重写 Results | 保留数值、分析集和不确定性 | 加入解释 |
| 重写 Discussion | 区分结果解释、外部证据和限制 | 把相关性升为因果 |
| 需要个人语气 | 先使用用户指定的 tone profile；没有时调用 rw-phd-tone 并提供样本 | 静默选择语气来源 |

## 核心规则

- 写作从章节功能和主张开始，不从漂亮句子开始。
- 章节编号不是写作功能。根据当前内容在研究论证中的作用判断。
- 每个部分都要完成自己的功能；解释、重要性和研究问题连接只在该功能需要时加入。
- 读者不能承担关键推理。需要读者知道的前提、解释和后果要在证据允许的范围内写明。
- 每个事实性主张都要有用户材料或可核验来源。
- 来源支持范围小于句子范围时，收窄句子。
- 结果、解释、推断和建议要使用不同的证据强度。
- Introduction 建立问题和缺口，不能提前假定研究结论。
- Methods 说明做了什么和为何这样做，不能用结果合理化方法。
- Results 报告预定和实际分析，不用 Discussion 语言解释。
- Discussion 先回答研究问题，再比较证据、解释限制和外推边界。
- 报告规范是最低透明度检查，不是段落模板。
- 用户修改要求优先保留原判断、证据和限制。
- 没有作者语料时使用克制的学术注册，不冒充个人指纹。
- 用户已指定可用的持久语气 profile 或本地 tone skill 时，先使用该 profile；没有时再由 rw-phd-tone 从本轮样本建立临时规则。
- 公开正文不得包含搜索日志、claim audit 路径、draft-status note 或其他内部交接说明。
- 引用、作者贡献、利益冲突和 AI 使用按目标期刊当前规则核验。
