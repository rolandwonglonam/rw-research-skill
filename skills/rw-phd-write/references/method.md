# RW PhD Write 方法

## 输入门

- 保留用户原问题和原始材料。
- 确认交付物、研究类型、可用证据和不能越过的范围。
- 缺少会改变判断的信息时，只追问必要项；其余内容继续处理。

## 执行阶段

1. 先建立 research output profile，记录机构、文稿类型、读者、研究问题、语言区域、引用格式、导师或审稿反馈、正向样本、输出格式和公开边界。
2. 依据内容和上下文判断文稿、章节、小节和段落的写作功能，不从章节编号推断功能。
3. 为当前功能确定解释义务：需要哪些证据、解释、重要性、研究问题连接和限制；同时标出不应出现在当前部分的内容。
4. 建立 function-to-claim-to-source 表，区分写作功能、目标主张、证据、作者解释、当前推断、重要性、研究问题连接和未知事项。
5. 按功能组织论证移动和段落顺序，先写论点和证据骨架，再写正文。
6. 确定完整论证单位。导师或审稿人只标出一个词句时，同时检查该句的前提、后果和相邻句；不要把批注框选范围误当作内容边界。
7. 运行概念闭合检查。对当前功能需要承担的命题，确认主体、对象、行动、发生环节、作用关系、适用边界和当前意义已经明说；不相关的项目不机械补入。
8. 运行读者推断检查，补足有材料支持的关键连接；证据不足时标记作者判断或证据缺口。
9. 按研究类型检查相应报告规范、方法透明度和披露事项。
10. 运行来源、具体性、因果强度、功能完成度、概念闭合、前后一致性、作者语气和修改范围检查，再把成稿交给引用核验和交付质检。

## 证据门

- 每项事实能回到用户材料或可核验公开来源。
- 推断标记为推断，缺失标记为缺失，不用常识补成已知事实。
- 当前信息在使用当日核验；稳定方法使用 Skill 内的摘要。
- 报告规范、偏倚工具和研究设计各自承担不同功能。
- 章节编号不能替代功能判断。
- 解释义务随功能变化，不把 evidence、interpretation、why it matters 和 research-question link 机械加入每一段。
- 关键逻辑不能只存在于作者脑中；无法由现有证据支持的连接保持为缺口。
- 抽象关系词不能充当解释的终点。使用 route、option、factor、decision、process 或 implementation 时，要让读者知道该词具体指向谁、什么对象和什么行动。
- 概念闭合以“不再需要猜关键关系”为停止条件，不以篇幅或句数为目标。

## 输出契约

- 章节、段落或论证骨架。
- research output profile、function-to-claim-to-source 表和引用缺口。
- 写作功能判断、解释义务、概念闭合断点、读者推断断点和未完成项。
- 修改说明、未解决问题和下游核验状态；按任务规模只展开必要部分。

输出同时标记：已确认事实、当前判断、来源、未知事项、停止条件和下一步。

## 停止条件

- 没有来源时，不生成事实性引用或伪造 DOI。
- 没有研究结果时，不写成研究已完成。
- 没有证据支持重要性或研究问题连接时，标记为作者判断或证据缺口，不生成通用意义句。
- 不为流畅删除限制、反例和不确定性。
- 不把 Skill 输出当成作者最终责任或期刊合规确认。
