# 科研写作功能和解释义务

## 判断原则

- 先判断当前部分在研究论证中做什么，再决定怎样写。
- 章节编号、标题和学校模板只能提供线索，不能单独决定功能。
- 同一段可以承担一个主要功能和一个次要功能；先保证主要功能完成。
- 每个部分都要让读者知道它为什么出现在这里，但不要求每段使用同一结构。

## 功能与义务

| 写作功能 | 通常需要完成 | 不要强行加入 |
|---|---|---|
| 建立研究问题 | 问题范围、受影响对象、已有证据、后果和研究入口 | 尚未得到的研究结论 |
| 综合文献 | 具体发现、可比性、共同点、冲突、作者综合判断和边界 | 按论文顺序逐篇复述 |
| 界定研究缺口 | 已知、未知、未知造成的限制、研究问题如何回应 | 仅凭“没有搜到”宣布空白 |
| 说明方法选择 | 做了什么、为何适合研究问题、替代方案和限制 | 用研究结果倒推方法正确 |
| 报告结果 | 分析对象、数值、比较、不确定性和预定或实际分析 | Discussion 式意义解释 |
| 解释结果 | 主要发现、可能解释、外部证据、替代解释和边界 | 超出设计支持范围的因果主张 |
| 讨论影响 | 影响对象、会改变的判断或行动、适用条件和证据强度 | 通用的“具有重要意义” |
| 说明限制 | 限制来源、可能影响的结果、方向和不能判断之处 | 只列限制名称而不说明后果 |
| 回答研究问题 | 回答、证据范围、贡献、边界和未解决问题 | 引入正文未建立的新主张 |

## 解释义务触发

出现以下情况时，检查是否需要补写：

- 连续列出研究、数字或引用，但没有综合判断。
- 提出“重要”“不足”“存在差异”或“需要研究”，但没有说明具体后果。
- 从证据跳到研究问题，中间的推理只存在于作者脑中。
- 比较研究结果，却没有说明人群、测量、设计、时间或分析差异是否影响可比性。
- 使用关键概念、指代或缩写，但读者无法从上下文确定含义。
- 段落结束后，读者仍不知道本段对当前小节或研究问题有什么作用。

不触发机械扩写。先确认当前功能是否需要解释。例如 Results 可以需要结果层 takeaway，但不因此加入机制、实践意义或政策建议。

## 导师和审稿反馈分类

- `Tell me more`：检查缺少研究细节、作者解释、逻辑连接还是研究问题关系。
- `Expand a bit`：补当前功能所需的信息，不统一拉长整段。
- `Can you say it in a different way`：检查概念边界、主语、指代、前提和句间关系，再决定是否改写。
- `I don't understand`：定位读者被要求自行补出的推理，不只替换词语。
- 正向评价：提取具体发现、数字、研究细节、引用位置、作者解释和句间连接，保持已经成立的部分。

## 段落记录

按任务需要记录：

- `function`：当前主要写作功能。
- `takeaway`：读者读完应获得的判断。
- `evidence`：支持该判断的材料和来源。
- `interpretation`：证据允许作者说明什么。
- `why_it_matters`：它改变什么判断、行动或研究需要。
- `research_question_link`：它如何服务当前研究问题。
- `boundary`：不能从当前材料得出的内容。
- `status`：`complete`、`needs_detail`、`needs_interpretation`、`needs_link` 或 `blocked`。

不要求每个字段都进入公开正文。字段用于判断和修订。
