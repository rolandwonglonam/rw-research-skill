# 来源证据

## 定位要求

- Markdown 和文本：文件路径、标题或附近文本。
- Word 和 PDF：文件路径、页码或段落定位；解析器无法保留页码时说明。
- CSV 和 Excel：文件路径、工作表或行列定位；当前提取器不保留单元格位置时说明。
- 画像中的每个 `applied`、`articulated` 和 `exposed` 判断至少连接 1 个原文定位。

## 证据强度

1. 用户在交付物中实际应用方法。
2. 用户用自己的语言解释并界定限制。
3. 多份内容中持续表达同一判断。
4. 单份笔记中的摘录或重述。
5. 文件名、标签、收藏和单次提及。

第 4、5 级证据不单独支持 `applied`。

## 归属和版本

- 每条正向证据记录 `source_kind`、`authorship_confidence` 和 `currentness`。
- 外部论文、教材、他人 thesis、下载的 Skill 和模板不支持用户的 `applied` 判断。
- AI 参与不自动否定用户能力，但需要过程记录或用户角色说明。
- 当前文件登记、最新工作稿和明确的 canonical 文件优先于旧 proposal。
- 旧材料中的方法只支持历史接触或历史应用，不自动成为当前方案。
