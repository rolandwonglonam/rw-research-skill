# 扫描策略

## 范围

| 模式 | 根目录 | 使用情形 |
|---|---|---|
| `folder` | 用户指定的一个或多个目录 | 用户长期在固定位置保存内容 |
| `current` | 当前工作目录 | 用户在一个项目里学习或研究 |
| `user` | 当前环境可读的用户主目录 | 用户明确要求全量盘点 |

拒绝将操作系统根目录作为扫描根。运行环境限制可读范围时，报告实际覆盖范围。

## 两阶段扫描

1. `landscape` 只读取路径、扩展名、大小和修改时间，用于发现候选内容目录。
2. `scan` 只对最终选择的根目录提取正文并建立索引。
3. 用户说“只看这个文件夹”时，不扩大范围。
4. 指定文件夹和自动发现目录分别记录，不能把候选范围说成已经完成内容扫描。

## 默认排除

- 版本库、依赖包、虚拟环境、编译目录、缓存、回收站、操作系统库和应用程序目录。
- 隐藏目录，除了扫描根目录本身。
- `.env`、私钥、证书、密码库、凭据文件和常见密钥文件名。
- 索引自身的 `.rw-research/` 目录。
- 超过默认大小限制的文件。
- 符号链接指向的目录和文件。

## 提取状态

- `indexed`：已提取可用文本。
- `partial`：文件可读，但只提取了前部分文本。
- `unsupported`：识别到文件，当前没有可用提取器。
- `too_large`：文件超出大小限制。
- `sensitive`：文件类型或文件名命中敏感排除规则。
- `unreadable`：权限或解析错误阻止读取。

## 增量扫描

1. 比较文件大小和纳秒修改时间。
2. 未变更文件保留现有文本和状态。
3. 新增和修改文件重新提取。
4. 已删除文件从当前索引中移除。
5. 每次扫描写入时间、根目录和各状态计数。

## 内容最小化

- 索引保存用于本地检索的提取文本。
- Agent 只读与当前主题有关的命中和原文样本。
- 不将全量索引复制到对话、公开仓库或远程服务。
- 用户删除状态目录后，本 Skill 不保留另一份副本。

## 私有状态

- 扫描索引、manifest、研究画像和能力报告默认为 `private_local`。
- 不自动复制到 README、示例、测试、公开仓库、Release 或远程服务。
- 公开测试只使用合成材料，不使用真实文件名、路径、人物、机构、研究内容或能力判断。
- 用户没有明确要求时，不导出或分享本地状态。
