# RW Statistics Audit 案例和反例

## 案例 1

- 输入：检查 3 次独立实验、每次 3 个技术孔的 n 是否报告正确。
- 必须做：固定文稿、表格、图和分析说明的版本，列出主要判断、结局和分析对象。；把问题分为错误、信息不足、需统计人员复核和报告通过；没有数据时不声称完成重算。；分析单位、重复层级、结局和方法对应表。
- 不能做：依赖未提供的本地工作区内容；生成不存在的论文、数据、DOI、工具结果或期刊要求；越过停止条件：分析单位或重复层级不清时，不给统计报告通过状态。
- 下一步：rw-research-referee。
