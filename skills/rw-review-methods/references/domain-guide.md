# RW Review Methods 选择表

| 情形 | 执行动作 | 不要做 |
|---|---|---|
| 回答干预效果 | 使用系统综述方法并计划偏倚和确定性评价 | 只做文献盘点 |
| 描绘概念和证据范围 | 使用范围综述方法 | 输出效果结论 |
| 研究高度异质 | 预设分层或结构化叙述综合 | 只报告投票计数 |
| 纳入非随机研究 | 选择相符的偏倚工具并说明 estimand | 套用 RoB 2 |
| 协议已注册后改变方法 | 记录偏离和影响 | 覆盖旧版本 |
| 范围综述准备开始正式检索 | 先完成 OSF registration，并核验 ID、URL、时间戳和冻结文件 | 把内部签字当成注册，或先检索后补注册 |

## 核心规则

- 先选综述类型，再选报告规范和方法手册。
- PRISMA 是报告规范，不是完整的综述实施方法。
- 范围综述用于描绘概念、证据类型和缺口，不默认回答效果大小。
- 系统综述的纳排标准要在批量筛选前固定。
- 正式证据综合在正式检索前完成适用的协议注册；范围综述使用 OSF。
- 内部 protocol acceptance 不等于外部 registration。
- 搜索日期、平台、完整搜索式和结果数量必须保留。
- 双人筛选和冲突处理规则要在执行前确定。
- 风险偏倚评价要与研究设计和具体结果匹配。
- 研究质量、报告完整性和偏倚风险不能混成一个分数。
- 是否进行统计合并由可比性和问题决定，不由研究数量决定。
- 没有元分析时仍需预先定义叙述综合方法。
- 协议偏离要记录时间、理由和对结论的影响。
- 证据确定性评价要在结果层进行，并说明降级或升级理由。
