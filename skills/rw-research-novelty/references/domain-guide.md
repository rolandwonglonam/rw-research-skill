# RW Research Novelty 选择表

| 情形 | 执行动作 | 不要做 |
|---|---|---|
| 只有宽泛研究兴趣 | 先进入 rw-research-question | 直接生成创新点清单 |
| 缺少相关文献 | 先进入 rw-literature-discovery | 用常识补证据 |
| 论文尚未结构化 | 先进入 rw-paper-extractor 或 rw-evidence-map | 按标题印象比较 |
| 已有证据图、冲突或缺口 | 生成多个可追溯候选并分别筛选 | 把第一个想法当结论 |
| 候选方向需要落地 | 转入 rw-research-design | 把构想写成完整方案 |
| 准备公开声称创新 | 转入 rw-research-referee 并保留领域专家确认 | 使用世界首次等无法核验的表述 |

## 核心规则

- 文献 gap 是输入，不是创新点结论。
- 没有检索到研究不能直接解释为没有研究。
- 候选方向必须能回到具体证据节点、冲突或限制。
- 发散要受证据和研究问题约束，不能只做自由联想。
- 每个候选方向至少列出两个替代解释和一个反例。
- 把理论贡献、方法贡献、数据贡献、情境扩展、复现和实施价值分开记录。
- 只更换国家、人群或变量名称不自动构成创新。
- 使用既有概念或方法也可能有价值，但要说明新的用途和影响。
- 新颖性、重要性、严谨性和可行性分别判断，不能用一个总分遮盖短板。
- 可执行不等于有贡献，有贡献也不等于当前资源可执行。
- 创新状态在当前检索和领域核验完成前只能标为候选。
- 保留被排除方向、理由和触发重新评估的条件。
- 涉及临床、伦理、统计或领域判断时保留人工确认。
