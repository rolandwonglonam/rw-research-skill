# RW Paper Extractor 选择表

| 情形 | 执行动作 | 不要做 |
|---|---|---|
| 随机试验 | 增加分配、盲法、分析集、流失和 harms 字段 | 只提 P 值 |
| 观察研究 | 增加暴露、混杂、选择、缺失和敏感性分析字段 | 把调整结果当随机结果 |
| 系统综述 | 增加协议、检索、筛选、偏倚和综合字段 | 按原始研究 schema 提取 |
| 定性研究 | 增加取样、情境、研究者角色、分析过程和引文依据 | 强行量化主题 |
| 批量任务 | 先锁 schema 和冲突规则 | 中途静默改字段 |
| 单篇 PDF 精读 | 建立 Paper Case，审核章节和图表，再按 5 个阶段写作 | 从整篇 PDF 直接生成无 locator 的长报告 |
| 跨页表格 | 拼接阅读图，同时保留各页 bbox 和 locator | 把拼接图标成单一原始页 |
| 接入 LitNet | 先过 Claim Audit，再生成回写预览 | 直接写入已验证主张 |

## 核心规则

- 提取 schema 由后续用途决定，不能对所有论文使用同一张空表。
- 标题和摘要只能提供它们实际报告的信息。
- Methods、Results 和 Discussion 的陈述类型要分开。
- 作者解释不能写成测得结果。
- 没有正文时不能推断随机化、盲法、模型设定或缺失数据处理。
- 表图中的分母、单位、时间点和分析集必须一起提取。
- 调整后和未调整结果不能混写。
- 主要、次要、探索性和事后分析要区分。
- 未报告、零事件和不适用不是同一个值。
- 批量提取前先用两篇不同结构的论文试表。
- 机器提取的字段要保留置信度和复核状态。
- 关键字段修改时保留原值、修改值和理由。
- 来源、配置或上游产物改变后，下游阶段标记为 STALE。
